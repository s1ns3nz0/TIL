# 1. 주요 질문
## 1) ALB는 왜 Public Subnet에 존재하는가?
- ALB는 인터넷에서 들어오는 트래픽을 받아 처리해야 한다.
- 사용자 브라우저에서 발생한 요청이 인터넷을 통해 IGW(Internet Gateway)와 연결된 ALB로 유입된 후, 적절하게 Private Subnet의 웹 서버로 분배된다.

## 2) Application Server는 왜 Private Subnet에 있는가?
- 서버를 외부에서 접근할 수 있도록 배치하면, 불필요한 공격표면이 노출되기 때문이다.
- ALB를 통해서반 트래픽을 수용하고 외부에서는 서버의 존재, 관련 정보를 모르게 하도록 하여야 한다.

## 3) NAT Gateway가 필요한 이유?
- Private Subnet에서 외부로 나가는 트래픽을 제어하기 때문이다.
- 서버에 설치된 패키지 등에 업데이트가 필요한 경우 인터넷과 제한적인 통신이 필요함

## 4) RDS를 Public Subnet에 두면 안 되는 이유는?
- RDS는 AWS의 관리형 데이터베이스 서비스로, Public Subnet에 위치하게 되면 인터넷으로 데이터베이스가 노출됨
- 데이터베이스에 저장하고 있는 고객정보, 관리자 계정 정보 등 주요 정보가 유출될 수 있음

## 5) 보안 그룹은 어떤 방향으로 열려 있는가?
- 보안 그룹은 Allow만 명시적으로 가능하며, 언급하지 않으면 기본 Deny 설정임
- Outbound에 대해서는 모두 허용이며 반대로 Inbound는 아무런 규칙이 없음(기본 Deny) 상태
- 허용된 IP, Protocol에 대해서 Inbound 접근 통제를 수행 하도록 설정 필요
